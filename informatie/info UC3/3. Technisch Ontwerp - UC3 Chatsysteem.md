### 4.3 EF Core Configuratie (`AppDbContext.cs`)

```csharp
public class AppDbContext : IdentityDbContext<ApplicationUser>
{
    public DbSet<Message> Messages { get; set; }

    protected override void OnModelCreating(ModelBuilder builder)
    {
        base.OnModelCreating(builder);

        // Cascade delete: als gebruiker wordt verwijderd, worden ook zijn berichten verwijderd
        builder.Entity<Message>()
            .HasOne(m => m.Sender)
            .WithMany()
            .HasForeignKey(m => m.SenderId)
            .OnDelete(DeleteBehavior.Cascade);
    }
}
```

## 5. SignalR Protocol

### 5.1 Hub-eindpunt

| Attribuut | Waarde |
|-----------|--------|
| URL | `/chatHub` |
| Transport | WebSocket (primair), Long-polling (fallback) |
| Autorisatie | `[Authorize]` – alleen ingelogde gebruikers |
| UserIdentifier | `Context.UserIdentifier` = `ApplicationUser.Id` |

### 5.2 Client → Server methoden

| Methode | Parameters | Beschrijving |
|---------|-----------|-------------|
| `SendMessage` | `content: string` | Stuurt een chatbericht |

**SendMessage – server-side validatie:**
```
content leeg/whitespace → ArgumentException → ReceiveError naar Caller
content > 1000 tekens   → ArgumentException → ReceiveError naar Caller
senderId ontbreekt      → Log Warning + return (geen actie)
```

### 5.3 Server → Client events

| Event | Parameters | Beschrijving |
|-------|-----------|-------------|
| `ReceiveMessage` | `sender: string, content: string, time: string (HH:mm)` | Nieuw bericht uitgezonden naar alle clients |
| `ReceiveError` | `message: string` | Foutmelding teruggestuurd naar de verzendende client |

### 5.4 Verbindingsstatus (Vue-client)

```
Initieel        → connected = false
start()         → connected = true
onreconnecting  → connected = false
onreconnected   → connected = true
onclose         → connected = false
```

### 5.5 Content Security Policy voor WebSocket

```
connect-src 'self' ws: wss:
```
Dit is noodzakelijk omdat SignalR een WebSocket-verbinding opent, die anders geblokkeerd zou worden door de CSP.

## 6. Authenticatie & Autorisatie (Technisch)

### 6.1 Identity-configuratie (Program.cs)

```csharp
builder.Services.AddIdentity<ApplicationUser, IdentityRole>(options =>
{
    // Wachtwoordbeleid ASVS V2.1
    options.Password.RequireDigit = true;
    options.Password.RequireLowercase = true;
    options.Password.RequireUppercase = true;
    options.Password.RequireNonAlphanumeric = true;
    options.Password.RequiredLength = 8;

    // Lockout ASVS V2.2
    options.Lockout.MaxFailedAccessAttempts = 5;
    options.Lockout.DefaultLockoutTimeSpan = TimeSpan.FromMinutes(15);
    options.Lockout.AllowedForNewUsers = true;
})
.AddEntityFrameworkStores<AppDbContext>()
.AddDefaultTokenProviders();  // Vereist voor 2FA TOTP
```

### 6.2 Cookie-configuratie (ASVS V3.4)

```csharp
builder.Services.ConfigureApplicationCookie(options =>
{
    options.Cookie.HttpOnly = true;              // Geen JS-toegang
    options.Cookie.SecurePolicy = CookieSecurePolicy.Always;  // Alleen HTTPS
    options.Cookie.SameSite = SameSiteMode.Strict;  // CSRF-bescherming
    options.ExpireTimeSpan = TimeSpan.FromHours(8);
    options.LoginPath = "/Account/Login";
});
```

### 6.3 Two-Factor Authentication (TOTP)

**Activeringsflow:**
1. `_userManager.ResetAuthenticatorKeyAsync(user)` → genereert 20-byte geheimsleutel
2. `_userManager.GetAuthenticatorKeyAsync(user)` → leest Base32-geformatteerde sleutel
3. `FormatKey()` → formatteert in blokken van 4 voor leesbaarheid
4. `GenerateQrCodeUri()` → genereert `otpauth://totp/ShowcaseChat:{email}?secret=...&issuer=ShowcaseChat`
5. QRCoder → Base64-encoded PNG (`data:image/png;base64,...`)

**Verificatieflow:**
```csharp
var code = model.Code.Replace(" ", "").Replace("-", "");
var result = await _signInManager.TwoFactorAuthenticatorSignInAsync(
    code, model.RememberMe, rememberClient: false);
```
ASP.NET Core Identity verifieert intern de TOTP-code (±30 seconden window, RFC 6238).

### 6.4 Autorisatiematrix (technisch)

| Route | Attribute | Rol vereist |
|-------|-----------|-------------|
| `/Chat` | `[Authorize]` | Ingelogd (elke rol) |

## 7. Security Headers Middleware

```csharp
app.Use(async (context, next) =>
{
    context.Response.Headers.Append("X-Content-Type-Options", "nosniff");
    // Voorkomt MIME-sniffing aanvallen

    context.Response.Headers.Append("X-Frame-Options", "DENY");
    // Voorkomt clickjacking

    context.Response.Headers.Append("Referrer-Policy", "strict-origin-when-cross-origin");
    // Beperkt referrer-informatie bij cross-origin verzoeken

    context.Response.Headers.Append("Content-Security-Policy",
        "default-src 'self'; " +
        "script-src 'self' 'unsafe-inline'; " +     // 'unsafe-inline' nodig voor SignalR JS
        "style-src 'self' 'unsafe-inline'; " +
        "connect-src 'self' ws: wss:");             // ws:/wss: voor SignalR WebSocket
    await next();
});
```

**ASVS-mapping:**
| Header | ASVS | Beschermt tegen |
|--------|------|----------------|
| X-Content-Type-Options | V14.4.1 | MIME-type sniffing |
| X-Frame-Options | V14.4.2 | Clickjacking |
| Content-Security-Policy | V14.4.3 | XSS, data-injectie |
| Referrer-Policy | V14.4.5 | Informatielekkage |
| HSTS (productie) | V9.1.1 | SSL-stripping |

## 8. Input Validatie & XSS-bescherming

### 8.1 Server-side validatie

| Laag | Mechanisme | Wat wordt gevalideerd |
|------|-----------|----------------------|
| ViewModels | DataAnnotations (`[Required]`, `[EmailAddress]`, `[StringLength]`) | Formulierinvoer |
| MessageService | Handmatige checks | Content niet leeg, max 1000 tekens |
| ChatHub | senderId niet leeg | Context.UserIdentifier aanwezig |

### 8.2 XSS-bescherming

| Locatie | Mechanisme | Detail |
|---------|-----------|--------|
| Razor Views | Automatisch HTML-encoding | `@Model.Content` → `&lt;script&gt;` |
| Vue Template | `{{ msg.content }}` (text interpolation) | Geen HTML-rendering; altijd geëscaped |
| Vue | Geen `v-html` directief gebruikt | Voorkomt DOM-based XSS |
| EF Core | Parameterized queries | Voorkomt SQL-injectie |

### 8.3 CSRF-bescherming

Alle POST-endpoints hebben `[ValidateAntiForgeryToken]`:
```csharp
[HttpPost]
[ValidateAntiForgeryToken]
public async Task<IActionResult> Register(RegisterViewModel model) { ... }
```
Razor-formulieren genereren automatisch het `__RequestVerificationToken` verborgen veld.

## 10. Database Seeding (DbSeeder.cs)

```csharp
public static async Task SeedAsync(
    RoleManager<IdentityRole> roleManager,
    UserManager<ApplicationUser> userManager,
    IConfiguration config,
    ILogger logger)
{
    // 1. Rollen aanmaken
    foreach (var role in new[] { "Admin", "User" })
        if (!await roleManager.RoleExistsAsync(role))
            await roleManager.CreateAsync(new IdentityRole(role));

    // 2. Admin-gebruiker aanmaken (uit configuratie)
    string email    = config["Seeder:AdminEmail"]    ?? "admin@showcase.nl";
    string password = config["Seeder:AdminPassword"] ?? "Admin@123!";

    if (await userManager.FindByEmailAsync(email) == null)
    {
        var admin = new ApplicationUser { UserName = email, Email = email,
                                          DisplayName = "Administrator",
                                          EmailConfirmed = true };
        var result = await userManager.CreateAsync(admin, password);
        if (result.Succeeded)
            await userManager.AddToRoleAsync(admin, "Admin");
    }
}
```

**Configuratie (appsettings.json / user-secrets):**
```json
{
  "Seeder": {
    "AdminEmail": "admin@showcase.nl",
    "AdminPassword": "Admin@123!"
  }
}
```

> **Veiligheid:** In productie dienen `AdminEmail` en `AdminPassword` via `dotnet user-secrets` te worden ingesteld, zodat ze niet in de git-repository terechtkomen.

## 15. Bekende Beperkingen

| Beperking | Reden | Mogelijke oplossing |
|-----------|-------|---------------------|
| SQLite schaalt niet horizontaal | SQLite is single-file; geen load balancing | Migreren naar SQL Server/PostgreSQL |
| SignalR in-memory (geen backplane) | Berichten gaan verloren bij herstart | Redis-backplane toevoegen |
| Berichthistorie alleen bij laden | Vue-component laadt geen scroll-back | Paginering implementeren |
| Geen berichtverwijdering | Valt buiten MVP-scope | Delete-endpoint toevoegen aan IMessageService |
| 'unsafe-inline' in CSP | Nodig voor SignalR JS; verslechtert CSP-niveau | Nonces toevoegen voor betere CSP |

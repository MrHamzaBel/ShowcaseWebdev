## 1. Inleiding

Dit implementatieplan beschrijft de bouwvolgorde, mijlpalen en aanpak voor UC3 – het realtime chatsysteem. Het plan is gebaseerd op de requirements uit de Studentenhandleiding (US4–US6) en sluit aan op de al gebouwde UC1 (Profielpagina) en UC2 (Contactpagina).

UC3 voegt de volgende functionaliteit toe aan de Showcase-applicatie:
- Authenticatie & autorisatie met rollen (User / Admin)
- Two-Factor Authentication via TOTP
- Realtime chat via SignalR + Vue 3 (Vite)
- Admin-dashboard voor berichtbeheer
- xUnit unit-tests + Cypress E2E-tests
- Security hardening conform OWASP ASVS

## 5. Bouwvolgorde (stap voor stap)

### Fase 1 – Projectopzet & Database (Week 1)
1. `dotnet new mvc` → project aanmaken
2. NuGet-packages installeren: EF Core SQLite, Identity, SignalR, QRCoder
3. `AppDbContext.cs` aanmaken (IdentityDbContext + Messages DbSet)
4. `ApplicationUser.cs` uitbreiden met `DisplayName`
5. `Message.cs` model aanmaken
6. EF Core migratie: `dotnet ef migrations add InitialCreate`
7. `DbSeeder.cs` schrijven (Admin + User rollen, admin@showcase.nl)
8. `Program.cs` configureren: Identity, SQLite, cookie-security

### Fase 2 – Authenticatie & Autorisatie (Week 1–2)
9. `AccountController.cs` – Register (POST), Login (POST), Logout
10. Razor-views: Login.cshtml, Register.cshtml
11. Identity-wachtwoordbeleid testen (8 tekens, upper/lower/digit/special)
12. Account lockout na 5 mislukte pogingen instellen en testen
13. `Enable2FA.cshtml` + TOTP QR-code genereren (QRCoder)
14. `Verify2FA.cshtml` + `TwoFactorAuthenticatorSignInAsync` implementeren
15. Rol-autorisatie: `[Authorize]`, `[Authorize(Roles="Admin")]`

### Fase 3 – Chat backend (Week 2)
16. `IMessageService.cs` interface definiëren
17. `MessageService.cs` implementeren: GetAllMessagesAsync, GetMessagesByUserAsync, SendMessageAsync
18. Validatie in SendMessageAsync: null-check, lege string, max 1000 tekens, trim
19. `ChatHub.cs` schrijven: `[Authorize]`, `SendMessage`, `ReceiveMessage`, `ReceiveError`
20. `ChatController.cs`: GET Index view + noscript-fallback POST
21. `AdminController.cs`: alle berichten + filter per gebruiker

### Fase 4 – Vue 3 + Vite frontend (Week 2–3)
22. `npm init` in `wwwroot-src/` → Vite + Vue 3 + @microsoft/signalr
23. `vite.config.js` configureren: output naar `wwwroot/js/chat-app.js`
24. `ChatApp.vue` schrijven: SignalR-verbinding, ReceiveMessage, SendMessage, auto-reconnect
25. Verbindingsstatus-badge (groen/rood), foutmelding-alert, scroll-to-bottom
26. `npm run build` → bundle genereren
27. Vue-component mounten in `Chat/Index.cshtml` via data-attributen (hubUrl, username)
28. Noscript-fallback testen (form POST zonder JS)

### Fase 5 – Security hardening (Week 3)
29. Security headers middleware: X-Content-Type-Options, X-Frame-Options, Referrer-Policy, CSP
30. CSP testen: connect-src ws: wss: voor SignalR WebSocket
31. HSTS inschakelen in productie
32. Anti-forgery tokens verificeren op alle POST-acties
33. XSS-bescherming: Razor escaping verifiëren, `{{ }}` in Vue (geen v-html)
34. SQL-injectiecheck: EF Core parameterized queries bevestigen

### Fase 6 – Testen (Week 3–4)
35. `MessageServiceTests.cs` schrijven: 8 tests (4 happy path, 4 unhappy path)
36. xUnit in-memory database setup (unieke naam per test)
37. Moq voor ILogger-afhankelijkheid
38. `dotnet test` uitvoeren en slagen bevestigen
39. Cypress installeren + `cypress.config.js` configureren
40. E2E-tests schrijven: register-flow, login-flow, bericht sturen, admin-filter
41. Cypress CI-ready maken

### Fase 7 – Integratie in ShowcaseWebsite (Week 4)
42. UC3-code overzetten naar `ShowcaseWebsite/` (geïntegreerde oplossing)
43. ContactController + MailService samenvoegen
44. Home-controller uitbreiden met CV-data
45. Migraties aanpassen voor gecombineerd schema
46. Eindtest: volledige flow UC1 + UC2 + UC3

## 8. Risico's & Mitigaties

| Risico | Kans | Impact | Mitigatie |
|--------|------|--------|-----------|
| SignalR WebSocket geblokkeerd door proxy | Middel | Hoog | Long-polling als fallback ingesteld |
| Vite-build mislukt in CI | Laag | Middel | `npm ci` + lockfile in repo |
| 2FA TOTP niet synchroon met klok | Laag | Middel | TOTP-venster ±30 seconden |
| EF Core migratie conflict | Laag | Hoog | Altijd `dotnet ef migrations add` op schone branch |
| SQLite concurrent schrijven | Laag | Laag | WAL-mode overwegen bij hogere belasting |
| Admin seed wachtwoord in config | Middel | Hoog | User-secrets gebruiken in productie |
